Handling Exceptions

Handling exceptions can be very important, especially when there is user input. You can handle exceptions using the handler keyword.

var try = handler { 10 / 0 # This will raise an operation error. } print(try) # '<OperationException:Division by zero, 2:10>' # ╔═══════Title═══════╗ ╔═════Message════╗ tuple(try) # ('Operation Exception', 'Division by zero', XX, XX) # Line Number╚╝ ╚╝Column Number try.name # 'OperationException' try.msg # 'Division by zero' try.file # 'XXXXX.XXX' try.loc # '<file>' try.line # XX try.column # XX try.text # '10 / 0'


Ignoring Handlers

The panic built-in function (See Built-In Functions) will raise a PanicException which can not be handled. These are dangerous to use with modules and other files.

var try = handler { panic('This is a problem') }
